## Task 5: Normalize exit code patterns in CLI handler

**Files:**
- Modify: `src/cli/commands/adopt.ts`

**Context:** The CLI handler mixes `process.exit(N)` (early exits, success) with `process.exitCode = N; return` (error paths). The success path's `process.exit(0)` can truncate stdout buffering when piped. Normalizing to `process.exitCode` + `return` everywhere makes the handler consistent and avoids truncation risk.

- [ ] **Step 1: Read the handler to identify all exit points**

Read `src/cli/commands/adopt.ts` in full. Identify every `process.exit(...)` and `process.exitCode = ...` site.

Expected sites (verify against current code):
- Early exit: no project root → `process.exit(1)`
- Early exit: lock failure → `process.exit(3)`
- Error paths: `process.exitCode = N; return`
- Success: `process.exit(0)`

- [ ] **Step 2: Replace process.exit() with process.exitCode + return**

For each `process.exit(N)` call:

```ts
// BEFORE (early exit on missing project root):
```
