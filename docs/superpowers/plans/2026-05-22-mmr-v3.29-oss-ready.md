# MMR v3.29 OSS-Ready Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship T1-B (declarative `output_parser` object form with `unwrap-jsonpath` and `regex-findings` kinds) and T1-G (compensator opt-in to any configured channel by reference) so MMR can run fully without Anthropic CLI dependencies once a local channel is configured via v3.28's `extends:` machinery.

**Architecture:** `output_parser` becomes a `string | discriminated-union` schema; a parser factory at dispatch time builds a `Parser` either by name (string form, today's behavior) or by constructing one from the config object (`{ kind: 'unwrap-jsonpath', wrap, then? }` or `{ kind: 'regex-findings', pattern, fields }`). The compensator reads an optional `defaults.compensator: { channel, channel_focus_map? }` block, resolves the referenced channel from the loaded config, and dispatches it through the same subprocess path used for normal channels; when no block is configured, today's hardcoded `claude -p` compensator is preserved unchanged.

**Tech Stack:** TypeScript, Zod schemas (`z.union` with `z.discriminatedUnion` arm for the parser object form; `z.object().optional()` for the compensator block), vitest. No new runtime deps — the jsonpath subset (`$.foo`, `$.foo.bar`, `$.foo[0]`, `$.foo[0].bar`) is implemented inline.

**Depends on v3.28 (assumed merged):** `extends:` / `abstract:` on `ChannelConfigSchema` (T1-A), the loader's `extends:` resolution + deep merge + merged-result validation (§5 decision 3), and `mmr review --dry-run` (T1-F). The abstract-reference check in Task 16 uses the v3.28 `abstract: true` field.

**Out of scope:** The `doc-conformance` built-in MMR channel keeps its current behavior unchanged. HTTP channels (`kind: http`, T1-C) land in v3.30 — the compensator implementation in this plan dispatches through `dispatchChannel` (subprocess) only.

/**
 * Minimal jsonpath subset used by the `unwrap-jsonpath` parser kind.
 *
 * Supported forms:
 *   $                        — root
 *   $.foo                    — property
 *   $.foo.bar                — nested property
 *   $.foo[0]                 — array index
 *   $.foo[0].bar             — property of an indexed element
 *
 * Returns `undefined` when the path does not resolve. Throws only for
 * structural errors in the path itself (bad bracket, missing `$`).
 */
export function jsonpathGet(root: unknown, path: string): unknown {
  if (!path.startsWith('$')) {
    throw new Error(`jsonpath must start with $ (got: ${path})`)
  }
  if (path === '$') return root

  // Strip the leading `$` then tokenize.
  const rest = path.slice(1)

  // Tokens are either ".name" or "[N]". We walk the string left-to-right.
  let cursor: unknown = root
  let i = 0
  while (i < rest.length) {
    const ch = rest[i]
    if (ch === '.') {
      // Read identifier up to next `.` or `[`.
      let j = i + 1
      while (j < rest.length && rest[j] !== '.' && rest[j] !== '[') j++
      const name = rest.slice(i + 1, j)
      if (name.length === 0) {
        throw new Error(`jsonpath malformed empty property at offset ${i} (path: ${path})`)
      }
      if (cursor === null || typeof cursor !== 'object' || Array.isArray(cursor)) {
        return undefined
      }
      cursor = (cursor as Record<string, unknown>)[name]
      i = j
    } else if (ch === '[') {
      const end = rest.indexOf(']', i + 1)
      if (end === -1) {
        throw new Error(`jsonpath malformed: unclosed [ at offset ${i} (path: ${path})`)
      }
      const inside = rest.slice(i + 1, end)
      if (!/^\d+$/.test(inside)) {
        throw new Error(`jsonpath invalid index "${inside}" (only numeric indices supported) at offset ${i} (path: ${path})`)
      }
      const idx = Number.parseInt(inside, 10)
      if (!Array.isArray(cursor)) return undefined
      cursor = cursor[idx]
      i = end + 1
    } else {
      throw new Error(`jsonpath unexpected character "${ch}" at offset ${i} (path: ${path})`)
    }
    if (cursor === undefined) return undefined
  }
  return cursor
}
```

```typescript
export function buildParser(spec: OutputParserConfig): Parser {
  if (typeof spec === 'string') {
    return getParser(spec)
  }
  switch (spec.kind) {
    case 'unwrap-jsonpath':
      return makeUnwrapJsonpathParser(spec.wrap, spec.then ?? 'default')
    default:
      throw new Error(`Unsupported output_parser kind: ${(spec as { kind: string }).kind}`)
  }
}

function makeUnwrapJsonpathParser(wrap: string, thenName: string): Parser {
  return (raw: string): ParsedOutput => {
    let outer: unknown
    try {
      // Allow `default`-style fence-stripping on the envelope itself so that
      // an LLM wrapping its JSON in ```json fences is still handled.
      const stripped = stripMarkdownFences(raw)
      outer = JSON.parse(stripped)
    } catch (err) {
      const msg = err instanceof Error ? err.message : String(err)
      throw new Error(`unwrap-jsonpath: envelope is not valid JSON (${msg})`)
    }
    const extracted = jsonpathGet(outer, wrap)
    if (extracted === undefined) {
      throw new Error(`unwrap-jsonpath: path ${wrap} did not resolve in envelope`)
    }
    if (typeof extracted !== 'string') {
      throw new Error(
        `unwrap-jsonpath: extracted value at ${wrap} is not a string (got ${typeof extracted})`,
      )
    }
    const inner = getParser(thenName)
    return inner(extracted)
  }
}
```

```typescript
export function buildParser(spec: OutputParserConfig): Parser {
  if (typeof spec === 'string') {
    return getParser(spec)
  }
  switch (spec.kind) {
    case 'unwrap-jsonpath':
      return makeUnwrapJsonpathParser(spec.wrap, spec.then ?? 'default')
    case 'regex-findings':
      return makeRegexFindingsParser(spec.pattern, spec.fields)
    default: {
      const k = (spec as { kind: string }).kind
      throw new Error(`Unsupported output_parser kind: ${k}`)
    }
  }
}

interface RegexFindingsFields {
  severity?: number
  location: number
  description: number
  suggestion?: number
}

function makeRegexFindingsParser(pattern: string, fields: RegexFindingsFields): Parser {
  let re: RegExp
  try {
    // Multi-line + global so users can write line-anchored patterns and
    // iterate matches over a full LLM-output blob.
    re = new RegExp(pattern, 'gm')
  } catch (err) {
    const msg = err instanceof Error ? err.message : String(err)
    return () => {
      throw new Error(`regex-findings: invalid pattern (${msg})`)
    }
  }

  return (raw: string): ParsedOutput => {
    const findings = []
    re.lastIndex = 0
    let match: RegExpExecArray | null
    while ((match = re.exec(raw)) !== null) {
      const get = (idx: number | undefined): string | undefined =>
        idx === undefined ? undefined : match?.[idx]
      const candidate = {
        severity: get(fields.severity),
        location: get(fields.location) ?? 'unknown',
        description: get(fields.description) ?? '',
        suggestion: get(fields.suggestion) ?? '',
      }
      findings.push(validateFinding(candidate))
      // Guard against zero-width matches (e.g. pattern `^`) creating an
      // infinite loop on global regexes.
      if (match.index === re.lastIndex) re.lastIndex++
    }
    return {
      approved: findings.length === 0,
      findings,
      summary: `regex-findings: ${findings.length} match${findings.length === 1 ? '' : 'es'}`,
    }
  }
}

Channels emit reviewer output in different shapes. `output_parser` accepts either a built-in parser name (string form — `default`, `gemini`, `doc-conformance`) or a structured object that builds a parser at dispatch time.

### `unwrap-jsonpath` — extract the model's response from an envelope

channels:

channels:

    command: my-linter
    flags: ["--format", "pipe"]

      kind: regex-findings
      pattern: "^(P[0-3])\\|([^|]+)\\|(.+)$"
      fields:

        location: 2
        description: 3
        suggestion: 4   # optional

import type { OutputParserConfig } from '../config/schema.js'

export interface CompensatorDispatch {
  command: string
  flags: string[]
  env: Record<string, string>
  stderr: 'capture' | 'suppress' | 'passthrough'
  output_parser: string | OutputParserConfig
}

/**
 * Resolve the dispatch parameters for a compensating pass.
 *
 * When `defaults.compensator.channel` is set, look up that channel's
 * config and return its dispatch shape so the existing subprocess
 * machinery (dispatchChannel) can run it. When unset, return today's
 * hardcoded `claude -p --output-format json` defaults (back-compat).
 *
 * `originalChannel` is the name of the channel being compensated for —
 * used only by callers that need it for logging or to look up an
 * override in `channel_focus_map` (Task 13 wires that in).
 */
export function resolveCompensatorDispatch(
  config: MmrConfigParsed,
  _originalChannel: string,
): CompensatorDispatch {
  const compConfig = config.defaults.compensator
  if (!compConfig) {
    return {
      command: 'claude -p',
      flags: ['--output-format', 'json'],
      env: {},
      stderr: 'capture',
      output_parser: 'default',
    }
  }
  const channelName = compConfig.channel
  const channelConfig = config.channels[channelName]
  if (!channelConfig) {
    // Loader already validates this, but guard at runtime as defense in depth.
    throw new Error(`Compensator channel "${channelName}" not found in config`)
  }
  return {
